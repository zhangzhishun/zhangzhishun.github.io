5天搞定Solidity

总目标

  1. 掌握Solidity语法
  2. 掌握Openzeeplin的使用，以ERC721为例写出完整合约
  3. 掌握合约编写、测试、部署全过程

第一天（预计4h）

目标

  1. 准备开发常用工具
  2. 准备测试币

过程

  1. 常用工具：
    • Remix：一款在线写Solidity的IDE，需要掌握编译、测试、部署、合约交互、调试的功能。很适合作为入门的开发环境选择。在Remix运行几个Solidity样例：https://solidity-by-example.org/，这将有助于理解Solidity
    • MetaMask：钱包，可以作为浏览器插件安装。创建自己的钱包，完成基本的导出助记词、私钥操作、配置Mumbai测试网络
  2. 领取测试币，因为部署合约需要消耗测试币。以Polygon测试链Mumbai为例，领币网站：
    • https://fauceth.komputing.org/?chain=5
    • https://goerlifaucet.com/

第二天（预计6h）

目标

  1. 掌握Solidity语法

过程

  1. Solidity基础课程：https://www.youtube.com/watch?v=xv9OmztShIw&list=PLO5VPQH6OWdVQwpQfw9rZ67O6Pjfo6q-p

第三天（预计6h）

目标

  1. 掌握Openzeeplin提供的ERC规范，熟悉规范内都有哪些功能
  2. 熟练使用Openzeeplin进行NFT开发

过程

  1. 阅读OpenZeeplin 合约下的代码，包括但不限于以下内容：
    • token：目录包括ERC20、721、777、1155合约
    • security：这些合同旨在涵盖常见的安全实践
    • access：该目录提供了限制谁可以访问合约功能或何时可以访问的方法
    • poxy：这是一组低级合约，实现了具有和不具有可升级性的不同代理模式
    • crosschain：该目录提供构建块来提高智能合约的跨链
    • util：包含实用功能的杂项合约和库，可用于提高安全性、处理新数据类型或安全使用低级原语

第四天（预计4h）

目标

  1. 使用OpenZepplin实现ERC721合约代码
  2. 【可选】学习Hardhat，了解Hardhat部署合约过程
  3. 【可选】基于Hardhat编写测试用例

过程

  1. 使用OpenZepplin扩展我们的功能，继承合约：

    • token目录的ERC721：ERC721实现

    • token目录的ERC721Burnable：支持Burn

    • token目录的ERC721Pausable：支持暂停

    • security目录的ReentrancyGuard：重入攻击

    • access目录的AccessControlEnumerable：启用权限控制

      pragma solidity ^0.8.4;

import "@openzeppelin/contracts/token/ERC721/ERC721.sol";
import "@openzeppelin/contracts/token/ERC721/extensions/ERC721Burnable.sol";
import "@openzeppelin/contracts/token/ERC721/extensions/ERC721Pausable.sol";

import "@openzeppelin/contracts/security/ReentrancyGuard.sol";
import "@openzeppelin/contracts/access/AccessControlEnumerable.sol";


contract ContractName is
ERC721,
ERC721Burnable,
ERC721Pausable,
ReentrancyGuard,
AccessControlEnumerable {
...
}

  2. 根据业务写相关代码，例如我们需要自己实现token_id和ipfs_hash的映射关系，那么我们添加代码：

    // key: token_id value: ipfs hash
mapping(uint256 => string) private _tokenURIs;

function _setTokenURI(uint32 tokenId, string calldata uri) internal {
   require(_exists(tokenId), "ERC721Metadata: URI query for nonexistent token");
   _tokenURIs[tokenId] = uri;
}

function setTokenURI(uint32 tokenId, string calldata uri) public onlyMaintainer {
   _tokenURIs[tokenId] = uri;
}

/**
* @dev See {IERC721Metadata-tokenURI}.
*/
function tokenURI(uint256 tokenId) public view virtual override returns (string memory) {
   require(_exists(tokenId), "ERC721Metadata: URI query for nonexistent token");
   string memory base = _baseURI();
   return bytes(base).length > 0 ? string(abi.encodePacked(base, _tokenURIs[tokenId])) : "";
}

  3. 可按实际需求实现更多功能
